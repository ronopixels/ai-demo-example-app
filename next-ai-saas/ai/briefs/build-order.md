# Recommended build order (phases)

Aligned with the Nexsas master guide. Use as a **sequencing** checklist, not a single task.

## Phase 1 — Foundation

- Design tokens / `globals.css`
- `cn`, UI primitives: button, input, card, badge, tabs, accordion (as needed)
- Layout: main navbar, footer, mobile menu (when marketing shell exists)

## Phase 2 — Marketing sections

- Hero, logo strip, features, product preview, integrations strip, pricing, testimonials, FAQ, CTA

## Phase 3 — Core marketing pages

- Home v1, About, Features, Pricing, Blog + detail, Contact, FAQ, Terms, Privacy

## Phase 4 — Auth

- Sign in, Sign up, Forgot password (+ reset/verify later)

## Phase 5 — Dashboard shell

- Dashboard layout, sidebar, topbar, content area

## Phase 6 — Dashboard pages

- Overview, Analytics, AI assistant, Workflow automation, Billing, Settings profile

## Phase 7 — More demos & extras

- Home v2/v3, helpdesk-style demo, integrations pages, use cases, compare, testimonials page

## Phase 8 — Packaging

- Docs, previews, HTML sync, changelog, QA pass

---

**Rule:** One agent task should still target **one page or one section** — this list is for **human** roadmap planning.
